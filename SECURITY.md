# Security Policy

## Supported Versions

Currently, only the major version (v3.x) is actively supported with security updates. Older versions are no longer maintained for security patches.

| Version | Supported          |
| ------- | ------------------ |
| 3.x.x   | :white_check_mark: |
| < 3.0   | :x:                |

## Reporting a Vulnerability

Security is taken very seriously in this project. If you discover a security vulnerability, please **DO NOT** open a public issue on GitHub. Publicly disclosing a vulnerability can put users at risk before a patch is available.

Instead, please report it privately by sending an email to:
**turkeryakup05@gmail.com**

### What to include in your report:
* A detailed description of the vulnerability.
* Step-by-step instructions on how to reproduce the issue.
* Any potential impact or risk assessment you have identified.

### Response Timeline
I will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about the progress. Once a patch is developed and deployed, you will be credited for the discovery (if you wish) in the release notes.

Thank you for helping keep this project safe and secure!
